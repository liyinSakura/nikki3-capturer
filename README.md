# 👗 nikki3-capturer 
《奇迹暖暖》(Love Nikki) 衣柜数据自动化提取工具。支持 macOS 上的 iPhone 镜像自动同步翻页截图，或批量导入已有截图，并利用大模型（LLM）进行高精度的结构化 OCR 文本识别，一键导出你的奇迹大陆资产！

> 💡 **项目起源**：作者自用小工具，旨在解决游戏内衣服太多、官方无导出功能、输出适合导入配装器的文本。

## ✨ 特性 (Features)

- 📸 **自动化捕获**：完美适配 macOS 上的 **iPhone 镜像 (iPhone Mirroring)**，支持分类一键自动截图、精准裁剪。
- 📂 **灵活导入**：支持将现有的游戏截图批量放入指定目录，进行离线批量识别。
- 🤖 **大模型高精度识别**：采用大语言模型（LLM）驱动，本地完成，完全免费。
## 🛠️ 运行环境与要求 (Prerequisites)

由于本项目核心采用了大模型进行视觉识别，对运行环境有一定要求：


### 1. 硬件与捕获环境
- **操作系统**：macOS (推荐)
- **捕获设备**：支持 iPhone 镜像功能的 iPhone + Mac

### 2. 识别后端（大模型）要求
为了照顾不同用户的配置，项目支持以下两种识别模式（可在配置文件中切换）：
- **本地大模型模式 (Local LLM)**：需本地安装并运行大模型（如通过 `llama.cpp` 运行）。**建议 Apple Silicon (M1/M2/M3/M4) 或有独立显卡的用户使用。**
- **在线 API 模式 (Cloud API)**：*(规划中)* 支持配置第三方大模型 API（如 DeepSeek, OpenAI 等），无需本地显卡，低成本高速度。

### 3. 配置文件（可选）
本项目支持在根目录放置 `config.json` 来覆盖默认设置。你可以直接复制 `config.example.json` 并修改以下参数：
- `BASE_DIR`: 要扫描的截图目录
- `OUTPUT_BASE_DIR`: 输出结果和日志目录
- `API_BASE`: 大模型后端地址
- `MODEL`: 模型名称或路径
- `MIN_EXPECTED_COUNT`: 单张图片判断为有效结果的最小编号数量
- `TIMEOUT`: API 请求超时时长（秒）
- `IMAGE_EXTS`: 支持的图片扩展名列表
- `HIGH_PRIORITY`: 优先处理的目录类别（第一批处理）
- `LOW_PRIORITY`: 次优先处理的目录类别（最后处理）
